# Traffic Light Controller Design

A Logisim-based digital circuit design for an intelligent traffic light controller. The system manages traffic flow at a two-way intersection and handles pedestrian crossing requests using a Finite State Machine(FSM).

## Project Overview

The controller regulates traffic at the intersection of North-South and East-West roads, incorporating a pedestrian crossing cycle triggered by a push button.

### Key Components
* **FSM State Memory:** A subcircuit that determines the next state based on the current state, a timer counter, and the pedestrian request button latch.
* **Output Logic:** A combinational logic subcircuit that decodes the active state into signals for the traffic lights(Red, Yellow, Green) and the pedestrian walk indicator.
* **Controller Implementation:** Built using D Flip-Flops for state registers, a 4-bit counter for timing states, and basic logic gates.

# Circuit Components

### 1. Three D Flip-Flops(FF2, FF1, FF0)

These flip-flops represent the system's memory and store the current state of the traffic intersection.

### 2. 4-Bit Counter

The counter tracks time by incrementing with every clock pulse(CLK), ensuring synchronous operation of the system.

### 3. 4-Bit Comparator

The comparator acts as a timing trigger. It detects when a traffic light phase has lasted for 10 seconds and generates a completion signal.

### 4. Combinational Logic Blocks

Logic gates determine both the next state of the finite state machine and the output signals, such as traffic light indicators.

### 5. Pedestrian Memory Latch

A specialized circuit consisting of an OR gate and a D Flip-Flop. It stores pedestrian crossing requests until they can be serviced.

### 6. Seven LEDs

The LEDs provide visual output for the system:

* North-South Red
* North-South Yellow
* North-South Green
* East-West Red
* East-West Yellow
* East-West Green
* Pedestrian Walk Signal

# Memory and Timing Control

## Binary Timing Calculations

The project simulates a 10-second duration for the main traffic light phases using a 4-bit Counter and a 4-bit Comparator.

### Target Duration

10(decimal)

### Binary Representation

10(decimal) = 1010(binary)

### Detection Mechanism

The comparator continuously monitors the counter value. When the counter reaches the binary pattern **1010**, the comparator outputs a HIGH signal:

**T_Done = 1**

This signal indicates that the current phase has reached its required duration and can transition to the next state.

## Reset Logic

To ensure that the timer restarts for each new state, a feedback-based reset mechanism is used.

### Operation

1. When **T_Done** becomes HIGH, the FSM transitions to the next state.
2. The counter's **Maximum Value** attribute is set to **0xA**(decimal 10).
3. Once the counter reaches this value, it automatically resets to **0000**.

As a result, every traffic light phase lasts exactly the same amount of time.

# Pedestrian Signals and Advanced Latch Design

## The Problem: Transient Inputs

A common issue in digital systems is the handling of transient inputs. During development, we identified that if a pedestrian presses the crossing button while the system is in a vehicle-flow state, the request could be lost if it is not stored.

## The Solution: Pedestrian Memory Latch

To solve this problem, we implemented a memory latch using an OR gate and a D Flip-Flop.

### Structure

1. The pedestrian button is connected to the input of an OR gate.
2. The output of the D Flip-Flop is fed back into the OR gate.
3. Once the button is pressed, the signal is latched and remains HIGH even after the button is released.

This feedback loop ensures that every pedestrian request is remembered until the system can process it.

## Breaking the Loop 

The stored request must be cleared after the pedestrian crossing phase has been served.

This is achieved using the signal generated by **State 4(WALK)**:

1. When the WALK state becomes active, it resets the pedestrian flip-flop.
2. The stored request is cleared.
3. The system is then ready to accept the next pedestrian request.

## Running the Simulation

To run and test the circuit:
1. Open `comp1_design.circ` in [Logisim](http://www.cburch.com/logisim/).
2. Enable automatic clock ticks by selecting **Simulation > Ticks Enabled** (Ctrl + K) in the menu.
3. Use the **Poke Tool** (hand icon) to press the pedestrian button and trigger the crossing cycle.
4. Adjust simulation speed if needed under **Simulation > Tick Frequency**.
